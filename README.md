# 天正暖通风机盘管自动选型脚本

用于根据 Excel 计算书中的房间冷负荷、湿负荷和新风参数，自动完成风机盘管（FCU）组合选型，并输出选型汇总表和详细计算表。

这个项目的目标是把原本需要人工反复查表、核算的选型过程，整理成一个可重复执行的脚本或可执行程序。用户只需要准备指定格式的 Excel，运行程序后即可得到可直接用于校核和汇总的结果文件。

## 项目现状

当前仓库里存在主程序、打包产物和历史分析材料。经核对，真正用于生成最终结果的主源码是：

- `fcu_selection.py`：主程序，负责读取输入、完成焓湿计算、组合选型、导出 Excel。
- `fcu_selection.spec`：PyInstaller 打包配置，入口文件就是 `fcu_selection.py`。
- `FCU选型程序.exe`：与 `dist/fcu_selection.exe` 的 SHA256 完全一致，是同一个最终程序的重命名副本。


另外，已经验证过：

- `fcu_selection.py` 对示例输入 `计算书.xlsx` 的计算结果，与仓库中的 `FCU_选型结果.xlsx`、`FCU_详细计算.xlsx` 数据逐列一致。


## 功能概览

程序完成两部分工作：

1. 按房间逐条进行空调焓湿计算，求出送风量、回风量、新风比、FCU 冷量、AHU 冷量、FCU 显热等结果。
2. 按计算得到的冷量和回风量，从预设 FCU 型号库中自动搜索 1 到 3 台设备的可行组合，并输出前 3 个方案。

输出文件：

- `FCU_选型结果.xlsx`：适合直接查看和汇总的选型结果表。
- `FCU_详细计算.xlsx`：包含全部中间参数和计算过程的明细表。

## 计算原理

项目使用 `psychrolib` 进行湿空气物性计算，单位制为 SI。

### 1. 已知输入

对每个房间，程序读取：

- 室内冷负荷 `Q`，单位 `kW`
- 室内湿负荷 `W`，单位 `kg/h`
- 新风量 `Vf`，单位 `m³/h`
- 送风温差 `dT`
- 新风机出风相对湿度 `RH_L`
- 新风管道温升 `dT_rise`
- 室内设计工况 `tN / RH_N`
- 室外设计工况 `tW / RH_W`

### 2. 室内、室外状态点

先由室内外温湿度求出状态点：

- 室内点 `N`：`dN`、`hN`
- 室外点 `W`：`dW`、`hW`

并进一步得到室外空气相关参数：

- 湿空气密度 `ρW`
- 干空气密度 `ρda = 1 / vW`
- 湿空气质量比热 `cp_moist = 1.006 + 1.86 × dW`
- 有效体积比热 `ρ_cp = ρda × cp_moist`

### 3. 新风机出风点 L

程序假定新风机处理后的出风点 `L` 满足：

- `RH_L` 为给定值，默认 `90%`
- `hL = hN`

也就是“新风机出风点与室内点等焓”。代码通过二分法求解对应温度 `tL`，再得到：

- `dL`
- `hL`
- `tIn = tL + dT_rise`

其中 `tIn` 为新风经过管道温升后的入室温度。

### 4. 送风状态与热湿比

先计算热湿比：

```text
ε = Q × 3600 / W
```

送风温度取：

```text
ts = tN - dT
```

然后沿热湿比线求送风含湿量 `ds`，再得到送风焓值 `hs` 和焓差：

```text
Δh_SHF = hN - hs
```

### 5. 送风量与回风量

程序优先采用 SHF 线焓差法：

```text
Vs = Q × 3600 / (ρda × Δh_SHF)
Vr = Vs - Vf
新风比 = Vf / Vs × 100%
```

如果 `Δh_SHF` 极小导致数值不稳定，代码会退回经验公式：

```text
Vs = Q × 1000 / (0.35 × dT)
```

### 6. 各项负荷拆分

新风管道温升负荷：

```text
Q_rise = Vf × ρ_cp × dT_rise / 3600
```

FCU 总冷量：

```text
Q_fcu = Q + Q_rise
```

AHU 冷量：

```text
Q_ahu = Vf × ρda × (hW - hL) / 3600
```

室内显热：

```text
Q_room_sen = Q - W × 2500 / 3600
```

新风承担的显热：

```text
Q_oa_sen = Vf × ρ_cp × (tN - tIn) / 3600
```

FCU 需要承担的显热：

```text
Q_fcu_sen = Q_room_sen - Q_oa_sen
```

## FCU 选型逻辑

程序内置如下 FCU 型号库：

| 型号 | 风量 (m³/h) | 制冷量 (W) | 水阻 |
| --- | ---: | ---: | ---: |
| FP-34 | 340 | 1800 | 30 |
| FP-51 | 510 | 2700 | 30 |
| FP-68 | 680 | 3600 | 30 |
| FP-85 | 850 | 4500 | 30 |
| FP-102 | 1020 | 5400 | 40 |
| FP-136 | 1360 | 7200 | 40 |
| FP-170 | 1700 | 9000 | 40 |
| FP-204 | 2040 | 10800 | 40 |
| FP-238 | 2380 | 12600 | 50 |

选型时会枚举 1 台、2 台、3 台的所有允许重复组合，并筛选满足以下条件的方案：

- 组合总制冷量 `>= Q_fcu × 1000`
- 组合总风量 `>= Vr`

排序规则强调“静音优先”：

1. 先比较组合中的“最大单台风量”，越小越优先。
2. 若最大单台风量相同，再比较“总风量余量”，越小越优先。

最终输出：

- 最优静音方案
- 备选方案 1
- 备选方案 2

组合字符串示例：

```text
2×FP-68 + 1×FP-34
```

## 输入文件要求

当前代码对输入文件名和工作表名是固定的：

- 输入文件名：`计算书.xlsx`
- 工作表名：`Sheet1`

程序从 `Sheet1` 中按列读取数据，实际使用的是前 13 列：

| 列序号 | 含义 | 说明 |
| --- | --- | --- |
| A | 楼层 | 当单元格包含“层”时，会被记录为当前楼层 |
| B | 房间 | 房间名称，不能为空 |
| C | 面积m2 | 可选，用于结果展示 |
| D | 室内冷负荷(不含新风/全热)KW | 必填，且必须大于 0 |
| E | 室内湿负荷(不含新风)kg/h | 必填 |
| F | 新风量m3/h | 必填 |
| G | 送风温差 | 为空时默认 `8.0` |
| H | 新风机露点相对湿度 | 为空时默认 `90.0` |
| I | 新风管道温升 | 为空时默认 `2.0` |
| J | 室内状态点干球温度 | 为空时默认 `25.0` |
| K | 室内状态点相对湿度 | 为空时默认 `60.0` |
| L | 室外状态点干球温度 | 为空时默认 `34.7` |
| M | 室外状态点相对湿度 | 为空时默认 `54.6` |

数据读取规则：

- 行内只要房间名、冷负荷、湿负荷、新风量有效，就会参与计算。
- 楼层允许只在某一行出现一次，后续空白行会继承最近一次识别到的楼层。
- 第二行空白不会影响程序。

仓库中的 `计算书.xlsx` 就是一份可直接参考的模板样例。

## 输出文件说明

### `FCU_选型结果.xlsx`

面向使用者的汇总结果，主要包含：

- 房间基础信息
- 热湿比、送风量、回风量
- FCU 冷量、AHU 冷量、FCU 显热、管道温升负荷
- 最优静音方案和两个备选方案

该文件还会额外生成一个工作表：

- `公式说明`：简要列出程序中使用的核心公式。

### `FCU_详细计算.xlsx`

面向复核和调试，除了输入列外，还包含：

- 室内/室外状态点参数
- 新风机出风点参数
- 送风状态参数
- 风量结果
- 各类负荷拆分结果

## 使用方法

### 方式一：直接运行 exe

1. 准备好输入文件，并命名为 `计算书.xlsx`。
2. 将 `计算书.xlsx` 放在程序同目录下。
3. 双击运行 `FCU选型程序.exe`，或使用 `dist/fcu_selection.exe`。
4. 程序执行完成后，会在同目录生成：
   - `FCU_选型结果.xlsx`
   - `FCU_详细计算.xlsx`

### 方式二：运行 Python 脚本

建议使用 Python 3.12。

先安装依赖：

```bash
pip install -r requirements.txt
```

然后在项目目录执行：

```bash
python fcu_selection.py
```

## 打包方法

当前仓库已经包含 PyInstaller 配置文件：

```bash
pip install -r requirements-build.txt
pyinstaller fcu_selection.spec
```

打包完成后，生成的可执行文件位于：

```text
dist/fcu_selection.exe
```

## 已知限制

- 输入文件名和工作表名当前是写死的，不支持命令行参数或图形界面选择。
- FCU 型号库写在源码中，不是外部配置。
- 组合选型目前只枚举最多 3 台设备。
- 程序默认面向 Windows 本地 Excel 工作流。
